It handles the full lifecycle of member management, deposits, loans, and disbursements for Saccos, Chamas, welfare associations, and cooperatives — with dedicated admin and member self-service portals.

Key Contributions

Standing Order Engine V2 — Built from Scratch

Designed the complete V2 standing order architecture with source-resolved orchestration, execution pipeline, repository contracts with read/write separation, and full API endpoints. Built matching Vue frontend with Vuex store, settings CRUD, batch monitoring, and observability screens.

Backend + Frontend Architecture Design

Group Setup Wizard — Built from Scratch

Built a complete 5-step group setup wizard supporting Chamas, welfare associations, and cooperatives. Features dynamic checklists based on account type, auto-derived disbursement modes, auto-creation of deposit products per sub-account, and progress persistence with event-driven updates. Full PHPUnit test coverage.

12 Commits PHPUnit Tests

Loan Management System

End-to-end loan processing: maker-checker approval workflows with multi-level approval, loan appraisal, amortization schedule generation, collateral management, guarantee system, and loan exports. Frontend includes full application lifecycle UI with optimized API calls.

15+ Commits Maker-Checker

Dashboard & Analytics

Built admin dashboard statistics API with consolidated repository, DataSummarizerService with batched processing, and backfill command for historical data. Refactored frontend dashboards consuming optimized endpoints with significantly reduced API calls.

Data Pipeline ApexCharts

Registration & Member Management

Built Chama/Group registration with DRY principles, unified company+user registration, group invite system with SMS notifications, OTP verification, multi-step member onboarding (6 steps + invite flow), and section counter auto-numbering with concurrency-safe retry logic.

18 Commits SMS Integration

DevOps & Deployment

Configured multi-server deployment for SNB-Sacco and SNB-Jamii on the same physical server. Parameterized Docker Compose files, set up Bitbucket Pipelines CI/CD, path-based routing with Nginx reverse proxy, and custom force-deploy pipelines for any branch.
